## 4. HOW IT'S BUILT (the generators)

The site pages are **generated by Python scripts**, not hand-edited HTML. The scripts live
**outside the repo**, in the Claude skill folder: `/root/.claude/skills/pdf-lou/`.
`prices.json` in that folder is the **single source of truth for every price** on the site.

Generators (`tools/`):
- `build_landing.py` — home page (`index.html`)
- `build_catalog.py` — model catalogue (`modeles.html`); face-crop via OpenCV
- `make_models_en.py` — FR→EN for the catalogue (`models-en.html`)
- `build_grille_web.py` — FR online rate card (`grille.html`), reads `prices.json`
- `make_grille_en.py` — FR→EN + € → £ for the rate card (`grille-en.html`)
- `build_travail_web.py` — the work page, both languages (`travail.html`, `travail-en.html`)
- `build_brief_web.py` — the online brief, both languages (`brief.html`, `brief-en.html`)
- `make_devis_en.py` — EN simulator (`devis-en.html`) from the FR template

Templates (`templates/`): `simulateur-devis.html` (FR quote simulator source),
`grille-tarifaire.html` / `-en.html` (rate card PDFs), `brief-video.html` / `-en.html` (brief PDFs),
`lou-denim.css` (PDF stylesheet).

**PDFs** are built with WeasyPrint from the template HTML. The simulator (`devis.html`) is built
by injecting the logo into `templates/simulateur-devis.html`.

### Rebuild order (matters — the landing must be generated LAST, some scripts write `index.html`)
```
python3 tools/build_catalog.py "<MODEL CATALOG folder>" <out>   # writes index.html → then mv to modeles.html
python3 tools/make_models_en.py <out>
python3 tools/build_grille_web.py <out> "devis.html" "modeles.html"
python3 tools/make_grille_en.py <out>
# devis.html: inject logo into templates/simulateur-devis.html
python3 tools/make_devis_en.py <out>
python3 tools/build_brief_web.py <out>
python3 tools/build_travail_web.py <out>
python3 tools/build_landing.py <out>            # LAST — writes the real index.html
# PDFs:
weasyprint templates/grille-tarifaire.html    <out>/Grille_Tarifaire_LouDenim.pdf
weasyprint templates/grille-tarifaire-en.html <out>/Rate_Card_LouDenim.pdf
weasyprint templates/brief-video.html         <out>/Brief_Video_IA_LouDenim.pdf
weasyprint templates/brief-video-en.html      <out>/AI_Video_Brief_LouDenim.pdf
# then: git add -A && git commit && git push  (goes live in ~1 min)
```

**⚠️ f-string trap:** `build_landing.py`, `build_grille_web.py`, `build_brief_web.py` are Python
f-strings — any literal `{` or `}` in injected CSS/JS must be **doubled** (`{{` `}}`).
`build_catalog.py`, `build_travail_web.py`, `make_*` and the templates are plain strings (single braces).

## 5. DESIGN SYSTEM (locked)

- **Background:** white, everywhere.
- **Animated background:** the "comet on a dot-grid" — a `<canvas id="dotfx">` with a grid of dots
  and **two** light comets (one left, one right, offset) tracing random bezier paths; dots go black
  as a comet passes, fade back to grey. ~2 KB of JS, no video. Present on every page.
- **Accent colour: pink `#E7549F`, HOVER ONLY.** Pink never appears as permanent decoration —
  only on hover (cards, buttons, icons) — **except** the FR/EN language pills (pink outline, filled when active).
  Form fields keep a **blue** hover (data entry ≠ navigation), never pink.
- **Font:** Jost (Google Fonts), weights 300–600.
- **Logo:** 86px tall on every page. Logo click: inner pages → studio home (`index.html`);
  studio home logo → the main photography site `loudenim.com`.
- **Client logos:** a right-to-left seamless marquee, **colour** logos from `img/logos-color/`
  (grey versions exist in `img/logos/`), pauses on hover.
- **Footers:** every page footer has email (mailto) + phone (tel), both clickable, and 5 social
  icons — Instagram, TikTok, YouTube, X, LinkedIn.
- **Blush palette (PDFs & form fields):** `#FBF1EF` / `#F5E3DF` / `#EBC9C3`. Never let it drift grey/green.
- **Document title bands:** rounded pill (border-radius 999px web / 4mm PDF).

## 8. HOW TO MAKE COMMON CHANGES

- **Change a price:** edit `prices.json` → rebuild `grille.html` (+ `grille-en.html`), the two rate-card
  PDFs (templates are hand-kept in sync with prices.json), and the simulator constants in
  `templates/simulateur-devis.html` (+ `make_devis_en.py`). Then push.
- **Add a model:** Lou adds a folder inside her `MODEL CATALOG` → reconnect/stage the folder →
  rerun `build_catalog.py` + `make_models_en.py` → push. The public link never changes.
- **Add a video to "the work":** give the YouTube link, say 16:9 or vertical → add the ID to the
  `FILMS` or `SHORTS` list in `build_travail_web.py` → rebuild → push. (Landscape play inline with a
  fullscreen button; verticals open in a large centred lightbox player.)
- **Any text/wording tweak:** edit the relevant generator/template → rebuild that page → push.
- Always keep FR and EN in sync (the `make_*_en.py` scripts hold the FR→EN string maps —
  if you change a FR string, update the matching entry in the EN script or it won't translate).
